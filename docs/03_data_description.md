# Описание данных

Контракт входных данных для **обучения** (CSV со всеми колонками, включая `target_class`) и перечень полей, необходимых для **инференса** (без таргета и без id в модели). Машинно-проверяемые правила — в `app/data/schema.py`.

## 1. Назначение строки

Одна строка = **одна завершённая поездка** каршеринга с признаками пользователя, поездки, ТС и контекста на момент формирования выгрузки.

## 2. Полный список колонок (обучение)

Порядок в CSV не важен; имена — **точное совпадение** с нижним регистром, как в таблице.

### Идентификаторы (в модель не подаются)

| Колонка | Тип в pandas | Обязательность |
|---------|--------------|----------------|
| `user_id` | object (string) | Да |
| `trip_id` | object (string) | Да |
| `car_id` | object (string) | Да |

### Числовые признаки

| Колонка | Смысл | Допустимые значения (инженерное ограничение демо) |
|---------|--------|-----------------------------------------------------|
| `user_age` | Возраст | 16–90 (валидация API); в CSV — разумный диапазон |
| `driving_experience_years` | Стаж, лет | ≥ 0; допускаются NaN (imputation median) |
| `account_age_days` | Возраст аккаунта, дни | ≥ 0 |
| `payment_delay_count` | Задержки платежей | целое ≥ 0 |
| `previous_incidents_count` | Инциденты в прошлом | целое ≥ 0 |
| `rating` | Рейтинг | 0–5; допускаются NaN |
| `trip_duration_min` | Длительность | ≥ 0 |
| `trip_distance_km` | Дистанция | ≥ 0 |
| `average_speed_kmh` | Средняя скорость | ≥ 0; NaN возможны |
| `max_speed_kmh` | Макс. скорость | ≥ 0 |
| `start_hour` | Час старта | 0–23 |
| `day_of_week` | День недели | 0–6 |
| `trip_cost` | Стоимость | ≥ 0 |
| `late_finish_flag` | Просрочка завершения | 0 или 1 |
| `car_age_years` | Возраст ТС | ≥ 0 |
| `mileage_km` | Пробег | ≥ 0 |
| `fuel_or_charge_level_start` | Уровень топлива/заряда, % | 0–100 |
| `fuel_or_charge_level_end` | Конец поездки, % | 0–100 |
| `night_trip_flag` | Ночная поездка | 0 или 1 |
| `weekend_flag` | Выходной | 0 или 1 |

### Категориальные признаки

| Колонка | Демо-допустимые значения (strict-режим в `prepare_data`) |
|---------|----------------------------------------------------------|
| `tariff_type` | `economy`, `standard`, `premium` |
| `trip_start_zone`, `trip_end_zone` | `north`, `south`, `east`, `west`, `center` |
| `car_category` | `compact`, `sedan`, `suv`, `van` |
| `weather_condition` | `clear`, `rain`, `snow`, `fog` |
| `traffic_level` | `low`, `medium`, `high` |
| `city_zone_risk` | `low`, `medium`, `high` |

На проде справочники расширяются; тогда отключают strict-проверку или обновляют `CATEGORICAL_ALLOWED` в `app/data/schema.py`.

### Целевая переменная

| Колонка | Значения |
|---------|----------|
| `target_class` | `0` или `1` (целое) |

## 3. Инференс (API / batch JSON)

Подаваемые поля — **все числовые и категориальные признаки из разделов выше**, без `user_id`, `trip_id`, `car_id`, без `target_class`. Схема запроса: `app/core/schemas.py`.

## 4. Источники данных (прод, ориентир)

| Блок | Вероятный источник |
|------|-------------------|
| Пользователь | CRM, биллинг, история инцидентов |
| Поездка | Телематика, события бронирования, тарификация |
| ТС | Fleet / asset management |
| Контекст | Погодные API, дорожная аналитика, справочник зон |

## 5. Качество и типичные проблемы

- Пропуски в телеметрии и рейтинге — обрабатываются **только** через обученный imputer.
- Рассинхрон часовых поясов при `start_hour` — риск; в демо не моделируется.
- **Утечка:** в признаки нельзя включать поля, доступные только после решения модератора о «проблемности», если они не определены на ту же временную метку, что и остальной вектор.
- Дисбаланс классов — учитывается `pos_weight` в обучении (см. `05_metrics_and_validation.md`).

## 6. Версионирование контракта

Изменение набора колонок = новая версия датасета: обновить `build_features.py`, `schema.py`, генератор, Pydantic-схемы API и переобучить модель с новым препроцессором.
