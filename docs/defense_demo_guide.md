# Сценарий демонстрации на защите диплома

Краткое руководство для устной защиты проекта **«Разработка нейросетевой модели классификации для задач каршеринга»** с использованием веб-интерфейса Streamlit.

## Подготовка (до выхода к комиссии)

1. Установить зависимости: `pip install -r requirements.txt`.
2. Запустить интерфейс: `streamlit run app/ui/streamlit_app.py` (или `python scripts/launch_ui.py`).
3. Убедиться, что в `data/raw/` есть CSV (кнопка генерации демо на странице **«Данные»**).
4. При необходимости один раз **обучить модель** в разделе **«Обучение модели»**, чтобы на страницах метрик и прогноза были актуальные артефакты.

## Рекомендуемый порядок показа (10–15 минут)

| Шаг | Раздел интерфейса | На что сделать акцент |
|-----|-------------------|------------------------|
| 1 | **О проекте** | Название темы, что это программный продукт, а не только код; статус модели и артефактов. |
| 2 | **Постановка задачи** | Бинарная классификация, классы 0/1, входы/выходы, цель и задачи работы, практический смысл. |
| 3 | **Данные** | Источник (демо vs свой CSV), таблица признаков, баланс классов; честно сказать про синтетику. |
| 4 | **Предобработка данных** | Цепочка: без сырья в сеть; пропуски, one-hot, масштабирование; fit только на train. |
| 5 | **Обучение модели** | MLP, BCEWithLogitsLoss + pos_weight, AdamW, ранняя остановка; показать графики потерь после прогона или из истории. |
| 6 | **Результаты и метрики** | Accuracy не в одиночку; precision/recall/F1/ROC-AUC; матрица ошибок; false positive vs false negative для каршеринга. |
| 7 | **Демонстрация прогнозирования** | Готовые сценарии (что меняется от базы) и ручной ввод; таблица результатов внизу экрана. Тексты — `docs/defense_narrative.md`. |
| 8 | **Практическая ценность** | Где применять, ограничения демо, путь к реальным данным. |

## Типичные вопросы комиссии

- **Почему нейросеть, а не бустинг?** — Универсальность для табличного вектора после one-hot; в репозитории возможны сравнения с baselines (скрипты/документация).
- **Как формируется таргет в демо?** — Правила генератора; на проде — согласованная бизнес-разметка.
- **Как боретесь с дисбалансом?** — `pos_weight` в функции потерь, интерпретация precision/recall.
- **Где утечка данных?** — Сплит стратифицированный, препроцессор только на train; идентификаторы не в модель.
