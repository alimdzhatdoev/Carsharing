# Каршеринг: нейросетевая классификация риска поездки (дипломное приложение)

Демонстрационный **исследовательский программный продукт** по теме **«Разработка нейросетевой модели классификации для задач каршеринга»**: бинарная классификация **нормальная поездка (0)** vs **повышенный риск (1)**. Ядро — MLP в PyTorch и пайплайн в `app/`; **интерфейс Streamlit** выстроен как **последовательность этапов дипломного исследования** (постановка → данные → предобработка → обучение → метрики → прогноз → ценность), полностью на **русском языке** в пользовательской части.

**Полное описание устройства проекта (поток данных, модули, артефакты, UI, API, скрипты):** **[docs/PROJECT_GUIDE.md](docs/PROJECT_GUIDE.md)** — начните с этого файла, если нужно разобраться целиком.

**Сценарий устной защиты:** [docs/defense_demo_guide.md](docs/defense_demo_guide.md). **Тексты для доклада по экранам:** [docs/defense_narrative.md](docs/defense_narrative.md).

## Быстрый старт (интерфейс)

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app/ui/streamlit_app.py
```

Браузер: **http://127.0.0.1:8501** (порт можно изменить в подсказке Streamlit).

Альтернатива-обёртка из корня проекта:

```bash
python scripts/launch_ui.py
```

Рекомендуемый сценарий (как на защите):

1. **О проекте** — цель, статус, артефакты, шпаргалка «сценарий защиты».
2. **Постановка задачи** — классы, входы/выходы, цель и задачи ВКР.
3. **Данные** — демо или свой CSV, описание признаков, баланс классов.
4. **Предобработка** — наглядный пайплайн до входа в нейросеть.
5. **Обучение модели** — параметры, запуск, графики потерь на русском.
6. **Результаты и метрики** — интерпретация precision/recall, матрица ошибок (подписи на графике — на русском после переобучения).
7. **Демонстрация прогнозирования** — готовые сценарии, ручной ввод, таблица признаков и пояснение результата.
8. **Практическая ценность** и **Справка**.

Подробный порядок показа и ответы на типичные вопросы: **`docs/defense_demo_guide.md`**.

**Режим данных:** по умолчанию допускается работа на **синтетическом демо-наборе**; переход на реальные выгрузки описан в разделе «Практическая ценность» интерфейса и в документации по схеме данных.

## Архитектура

| Слой | Назначение |
|------|------------|
| `app/data/` | Загрузка CSV, валидация, сплиты, препроцессинг, датасет |
| `app/features/` | Списки признаков и таргета (single source of truth) |
| `app/models/` | Сеть, обучение, оценка, predict |
| `app/services/` | Оркестрация для UI/API: данные, обучение, оценка, статус, предсказания |
| `app/api/` | FastAPI (опционально, помимо UI) |
| `app/ui/` | Streamlit: только вызовы сервисов и отображение |
| `scripts/` | Тонкие entrypoint’ы и утилиты CLI |

Интерфейс **не дублирует** логику обучения и инференса — вызывает функции из `app/services/`.

## Разделы интерфейса (все пользовательские тексты — на русском)

| Раздел | Содержание |
|--------|------------|
| **О проекте** | Тема ВКР, предметная область, статус системы, артефакты, метрики, инструкция и блок «сценарий защиты» |
| **Постановка задачи** | Цель, задачи, объект/предмет исследования, классы, вход/выход, практический смысл |
| **Данные** | Источник данных, генерация/загрузка CSV, описание признаков, полная таблица, графики по столбцам, баланс классов |
| **Предобработка** | Пошаговое объяснение пайплайна sklearn → вход MLP |
| **Обучение модели** | Описание MLP и функции потерь, параметры, запуск, графики потерь и F1 (подписи на русском) |
| **Результаты и метрики** | Метрики, ROC/PR, гистограммы по матрице ошибок, FP/FN, пояснения для защиты, отчёты sklearn/md в expander |
| **Демонстрация прогнозирования** | Сценарии, ручной ввод, справка по условным весам признаков, результат и текст «почему так» |
| **Практическая ценность** | Применение, ограничения, переход на реальные данные |
| **Справка** | Команды запуска, пошаговая инструкция, сценарий защиты |

## Установка зависимостей

```bash
pip install -r requirements.txt
```

Для разработки (опционально): `pip install -e ".[dev]"` — см. `pyproject.toml`.

## Данные (помимо UI)

Генерация демо-CSV вручную:

```bash
python scripts/generate_demo_data.py --rows 8000 --output data/raw/trips_demo.csv
```

Проверка контракта:

```bash
python scripts/prepare_data.py --input data/raw/trips_demo.csv --strict --write-profile
```

Контракт колонок: `docs/03_data_description.md`.

## Обучение и оценка (CLI, опционально)

```bash
python scripts/train.py
python scripts/evaluate.py
```

Артефакты: `artifacts/models/`, `artifacts/encoders/`, `artifacts/metrics/`, `artifacts/reports/`.

## API (FastAPI)

Локально:

```bash
uvicorn app.api.main:app --reload --host 0.0.0.0 --port 8000
```

- Swagger: http://127.0.0.1:8000/docs  
- Подробности: `docs/06_api_usage.md`

## Docker

**Приоритет — контейнер с UI** (`Dockerfile.ui`).

Только Streamlit (порт **8501**):

```bash
docker compose build ui
docker compose up ui
```

Откройте http://localhost:8501. Рекомендуется смонтировать локальные `data/` и `artifacts/` (уже в `docker-compose.yml` для сервиса `ui`), чтобы сохранять данные и веса между перезапусками.

**UI + API** (8501 и 8000):

```bash
docker compose --profile api up --build
```

Только API (без UI):

```bash
docker compose --profile api up api
```

Классический образ API: `Dockerfile` (корень), сервис `api` в compose.
